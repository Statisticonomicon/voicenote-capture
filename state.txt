PROCESSING PROTOCOL (mobile ProcessWorker) — current
====================================================================
ProcessWorker is a CoroutineWorker. Real (non-mock) flow against the
endpoint BASE url configured in Settings (e.g.
http://tailscale-node.tail05233a.ts.net:8457):
  1. POST audio (multipart field "audio") -> {base}/upload -> {job_id}
  2. poll GET {base}/status/{job_id}  (queued/loading_model/transcribing)
     until "done" or "error"
  3. on done: GET {base}/download/{job_id}?format=plain -> transcript
  4. write transcript to the vault (SAF), unchanged
  5. on error: log "error" field, clear job_id, retry
Robustness: per-call HTTP timeouts; job_id persisted per audio path so a
retry RESUMES polling instead of re-uploading; per-execution poll budget
under WorkManager's ~10 min cap; runaway cap MAX_RUN_ATTEMPTS=5 via
runAttemptCount (then log + clear job_id + Result.failure()). Mock mode
(default) skips network and writes canned text. No new deps (HttpURLConnection
+ platform org.json).

Cleartext: targetSdk 34 blocks http:// by default. res/xml/
network_security_config.xml permits cleartext ONLY for
tailscale-node.tail05233a.ts.net (compiled into the APK); all other hosts
stay HTTPS-only. Referenced from <application android:networkSecurityConfig>.
