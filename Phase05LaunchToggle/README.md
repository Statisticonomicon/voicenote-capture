# Phase 0.5 - Launch-Toggle Spike

Part of the Voice Note Capture project. Prototype / gating spike.

## What it does
Tests whether the app can be driven through the Android LAUNCH path instead of
hardware key events (which Phase 0 proved are not delivered to a foreground Wear
app). With launchMode=singleTask, the first launch starts "recording" via
onCreate(), and each re-launch toggles via onNewIntent() - the behaviour a
hardware button assigned to the app would produce when pressed repeatedly.

## Background: Phase 0 result
adb stem keyevents 264-267 did not reach onKeyDown; 264 went to Recents and
265-267 fired launch intents ("Starting..."). Conclusion: Wear OS reserves the
hardware buttons; apps cannot intercept them as key presses. Hence this
launch-path approach.

## Install / run
See HOW_TO_RUN.md. In short: open in Android Studio (Panda 4), sync, run on a
Wear OS emulator, then `adb shell am start -n com.notaricus.phase05/.MainActivity`
repeatedly and confirm the status toggles via onNewIntent.

## Features
- onCreate() -> START (first launch).
- onNewIntent() -> toggle START/STOP (re-launch; the mechanism under test).
- Distinct start/stop haptics (no-op on emulators without a vibrator).
- On-screen tap fallback toggle (guaranteed control path).
- Standalone Wear app; singleTask launch mode.

## Configuration options
None. minSdk 30, target/compileSdk 34, applicationId com.notaricus.phase05.
Toolchain: AGP 8.13.0 / Kotlin 2.1.20 / Gradle 8.14 / Java 21 (Panda 4 baseline).

## File structure
- settings.gradle.kts, build.gradle.kts, gradle.properties - Gradle setup.
- gradle/wrapper/gradle-wrapper.properties - wrapper config (jar regenerated by Studio).
- app/build.gradle.kts - module config.
- app/src/main/AndroidManifest.xml - watch feature, VIBRATE, singleTask launcher activity.
- app/src/main/java/com/notaricus/phase05/MainActivity.kt - onCreate/onNewIntent toggle logic.
- app/src/main/res/layout/activity_main.xml - status, fallback button, log.
- app/src/main/res/values/strings.xml - strings.
- app/src/main/res/drawable + mipmap-anydpi-v26 - adaptive launcher icon.
- HOW_TO_RUN.md - run + test steps and how to read the result.

## Troubleshooting
- Android Studio may prompt to upgrade AGP/Gradle on first open - accept it.
  Pinned to Panda 4 baseline (AGP 8.13.0 / Kotlin 2.1.20 / Gradle 8.14 / Java 21).
- No gradlew wrapper jar bundled; Studio regenerates it on import, or run
  `gradle wrapper` once.
- Vibration is a no-op on emulators without a virtual vibrator - expected; the
  call path still runs (check Logcat).
