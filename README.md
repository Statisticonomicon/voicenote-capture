# Voice Note Capture

Wrist-driven, eyes-free voice note capture. Press a button on the watch, speak,
press to stop; the audio is handed to the phone, sent to a processing endpoint
(your home server by default), and the returned text is written into your
Obsidian vault. Prototype (Phase 1).

## What it does
- **Watch (`:wear`)**: single-button launch-toggle activation (confirmed in Phase
  0.5) - first press starts recording, each subsequent press toggles stop/start;
  distinct start/stop haptics; screen stays off. Recording runs in a microphone
  foreground service (mandatory on Android 14+). The finished file is sent to the
  phone over the Wear Data Layer (ChannelClient). On-screen tap is the guaranteed
  fallback control.
- **Phone (`:mobile`)**: receives the audio, saves a raw copy to a chosen folder,
  then (via WorkManager, with retry) POSTs it to the configured endpoint and writes
  the returned text as a Markdown note into the chosen Obsidian vault folder.
- **Endpoint**: a black box returning text. Default is your home server over
  Tailscale (transcribe + summarise + any post-processing). A cloud option and an
  in-app mock mode are supported. Server internals are out of scope.

## Install / run / test
See HOW_TO_RUN.md. Three independent test slices: watch alone, phone alone (with a
manual audio-import path so no watch is needed), and the watch<->phone link via
paired emulators. Built for Android Studio Panda 4.

## Features
- Launch-toggle activation + on-screen tap fallback.
- Microphone foreground service with correct Android 14+ type/permissions.
- Distinct start/stop haptics; screen-off capture.
- Optional max-duration auto-stop (OFF by default; constant in RecordingService).
- Data Layer file transfer (ChannelClient) keyed by a shared capability.
- Phone: SAF folder selection for raw + vault (no broad storage permission).
- WorkManager upload queue with retry/backoff.
- Configurable endpoint + auth token; in-app mock mode (default ON).
- Manual "import audio" test path on the phone.

## Configuration options (phone settings)
- Processing endpoint URL; auth token (optional).
- Mock mode (default ON).
- Raw-audio folder (SAF); Obsidian vault folder (SAF).
- (Watch) max-duration auto-stop: currently a constant, OFF by default; promoted
  to a synced setting in a later phase.

## File structure
- settings.gradle.kts, build.gradle.kts, gradle.properties - root Gradle.
- gradle/libs.versions.toml - version catalogue (the keep-current single source).
- gradle/wrapper/ - wrapper config (jar regenerated by Studio).
- wear/ - watch module:
  - WearMainActivity.kt (launch-toggle, permissions, haptics, service control)
  - RecordingService.kt (mic foreground service, MediaRecorder, auto-stop)
  - WearTransfer.kt (Data Layer ChannelClient send)
  - res/ (layout, strings, icon), AndroidManifest.xml
- mobile/ - phone module:
  - SettingsActivity.kt (settings + SAF pickers + manual import test)
  - Settings.kt (prefs + SAF tree URIs)
  - PhoneListenerService.kt (receives files over Data Layer)
  - ProcessWorker.kt (POST to endpoint / mock, write note to vault)
  - res/ (layout, strings, theme, icon, wear_capabilities.xml), AndroidManifest.xml
- tools/mock_endpoint.py - optional local mock server for network testing.
- HOW_TO_RUN.md, USER_GUIDE.md.
- docs/spikes/ - notes from earlier prototype spikes (superseded by Phase 1,
  kept for history):
  - PHASE0_README.md - Phase 0 button-event spike (proved hardware stem keys
    do not reach the app as key events).
  - PHASE05_HOW_TO_RUN.md - Phase 0.5 launch-toggle spike (the activation
    workaround now used by the watch module).

## Troubleshooting
- AGP upgrade prompt on first open: accept it. Missing wrapper jar: let Studio
  regenerate or run `gradle wrapper`.
- Verify `desugarJdkLibs` in the catalogue against Maven Central; bump if newer.
- Watch logs "No reachable phone node": expected when no phone is paired (Slice A).
- Vault note not written: confirm the vault folder is set in phone settings (SAF
  permission persisted) and Mock mode/endpoint are configured as intended.
- Mic FGS won't start: it must be triggered from the foreground (the launch press),
  never the background - this is an Android 14+ while-in-use restriction, not a bug.
