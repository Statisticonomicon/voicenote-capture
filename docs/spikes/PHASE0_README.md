# Phase 0 — Button Event Spike

Part of the Voice Note Capture project. Prototype / gating spike.

## What it does
A minimal standalone Wear OS app whose only job is to answer the Phase 0 gating
question: do hardware button presses reach a foreground Wear app, and which
KEYCODE_STEM_* corresponds to which physical button?

It logs every key event (code + name) on screen and to Logcat, toggles a big
IDLE/ACTIVE status on each stem-key press, fires a distinct start/stop vibration,
and offers an on-screen "tap toggle" as a fallback control.

## Install / run
See HOW_TO_RUN.md. In short: open this folder in Android Studio, let it sync,
create a Wear OS emulator (API 34), run, then inject stem keys with adb and watch
the log.

## Features
- onKeyDown / onKeyUp capture of all key events, displayed and logged.
- Toggle on KEYCODE_STEM_PRIMARY / STEM_1 / STEM_2 / STEM_3.
- Distinct start/stop haptic patterns (no-op on emulators without a vibrator).
- On-screen tap fallback toggle.
- Standalone Wear app (no phone app needed for the spike).

## Configuration options
None at this stage. minSdk 30, targetSdk/compileSdk 34, applicationId
com.notaricus.phase0.

## File structure
- settings.gradle.kts, build.gradle.kts, gradle.properties — Gradle setup.
- gradle/wrapper/gradle-wrapper.properties — wrapper config (jar regenerated by Android Studio).
- app/build.gradle.kts — module config.
- app/src/main/AndroidManifest.xml — watch feature, VIBRATE, launcher activity.
- app/src/main/java/com/notaricus/phase0/MainActivity.kt — all spike logic.
- app/src/main/res/layout/activity_main.xml — status, fallback button, log.
- app/src/main/res/values/strings.xml — strings.
- app/src/main/res/drawable + mipmap-anydpi-v26 — adaptive launcher icon.
- HOW_TO_RUN.md — emulator + adb test steps and the acceptance check.

## Troubleshooting
- Android Studio may prompt to upgrade AGP/Gradle on first open — accept if offered.
  The pinned versions (AGP 8.5.2 / Kotlin 2.0.20 / Gradle 8.9) are a known-good
  baseline but may be older than your install.
- No gradlew wrapper jar is bundled; Android Studio regenerates it on import, or
  run `gradle wrapper` once in the folder.
- Vibration does nothing on emulators without a virtual vibrator — expected; the
  call path is still exercised (check Logcat).
- If stem-key injections do NOT appear in onKeyDown, that is a valid negative
  result: note it, use the tap path for Phase 1, and re-test on the OnePlus in
  Phase 2.
